# リスク管理設計

## 投入額計算（固定パーセントモデル）

全資産の X% をリスクにさらす方式を採用する。

### 計算式

```
リスク額 = 口座残高 × リスク率(%)
ポジションサイズ = リスク額 ÷ (エントリー価格 - SL価格)
```

### 例

- 口座残高: $10,000
- リスク率: 1%
- エントリー価格: $45,000
- SL価格: $44,500（$500 の差）

```
リスク額 = $10,000 × 0.01 = $100
ポジションサイズ = $100 ÷ $500 = 0.2 BTC
```

## SL / TP 初期設定

### SL（損切）設定

ATR ベースで動的に計算する：

```
SL幅 = ATR(14) × 係数（例: 1.5~2.0）
ロング SL = エントリー価格 - SL幅
ショート SL = エントリー価格 + SL幅
```

### TP（利確）設定

リスクリワード比 1:2 以上を基本とする：

```
TP幅 = SL幅 × RR比（例: 2.0）
ロング TP = エントリー価格 + TP幅
ショート TP = エントリー価格 - TP幅
```

TP はAIエージェントの判断により更新可能（継続判断時）。

## 手動トレーリングストップ

### 概要

Python 側の常駐プロセスとして実装し、Hyperliquid の組み込み機能に依存しない。

### ロジック

```
1分ごと（または価格更新ごと）に現在価格をチェック

ロングポジションの場合:
  現在価格が (最高値 - トレーリング幅) を下回った → SL 発動

ショートポジションの場合:
  現在価格が (最安値 + トレーリング幅) を上回った → SL 発動

SL 更新条件（有利な方向への移動）:
  ロング: 現在価格 > 記録済み最高値 → 最高値を更新し、SL を引き上げ
  ショート: 現在価格 < 記録済み最安値 → 最安値を更新し、SL を引き下げ
```

### トレーリング幅の設定

```
トレーリング幅 = ATR(14) × トレーリング係数（例: 1.0~1.5）
```

## 最大ホールド期間

- 最大 24時間でポジションを強制決済
- メインループ（15分ごと）で保有時間をチェック
- ホールド時間超過でリスクエージェントが「決済」を強く推奨

## Funding Rate リスク管理

- FR が一定以上（例: 0.1%/8h 超）の場合、ロングポジションを抑制
- FR が一定以下（例: -0.1%/8h 未満）の場合、ショートポジションを抑制
- FR の累積コストをリスクエージェントが考慮

## リスクパラメータ一覧

| パラメータ | デフォルト値 | 説明 |
|---|---|---|
| `RISK_PERCENT` | 1.0% | 1トレードあたりのリスク率 |
| `DEFAULT_LEVERAGE` | 5x | デフォルトレバレッジ |
| `SL_ATR_MULTIPLIER` | 1.5 | SL幅のATR乗数 |
| `TP_RR_RATIO` | 2.0 | リスクリワード比（TP設定用） |
| `TRAILING_ATR_MULTIPLIER` | 1.0 | トレーリングストップのATR乗数 |
| `MAX_HOLD_HOURS` | 24 | 最大ポジション保有時間 |
| `FR_LONG_THRESHOLD` | 0.001 | ロング抑制のFR閾値（8h） |
| `FR_SHORT_THRESHOLD` | -0.001 | ショート抑制のFR閾値（8h） |

これらのパラメータは WebUI の設定画面から変更可能にする。
