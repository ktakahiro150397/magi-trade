# プロジェクト概要 / Project Overview

## システム概要

Hyperliquid（分散型取引所）上で動作する、複数のAIエージェントによる合議制を用いた自動取引システム。

- 15分足のOHLCVデータ、Funding Rate（FR）、HLPデータを分析
- 複数の役割を持つAIエージェントが合議・議論し、エントリー/決済を判断
- Docker環境（自宅サーバー）で24時間365日稼働

## システム名

**Magi Trade** — 複数AIエージェントの合議 (MAGI System) にちなむ

## 判断ロジック概要

### ポジションなし時
| 判断 | 内容 |
|---|---|
| ロング | 買いポジションを開く |
| ショート | 売りポジションを開く |
| 見送り | エントリーしない |

### ポジションあり時
| 判断 | 内容 |
|---|---|
| 継続 | 現ポジションを保持（TP更新含む） |
| 決済 | ポジションをクローズ |

## 主要制約・方針

- **時間足:** 1m / 15m / 30m / 1h のデータを定期取得
- **AIへの入力:** 画像不可。RSI / ATR / EMA 等テクニカル指標のJSONデータのみ
- **最大ホールド期間:** 1日（24時間）
- **投入額計算:** 固定パーセントモデル（全資産の X% をリスクにさらす）
- **セキュリティ:** API Key / Secret は環境変数で管理、DBには保存しない

## 可視化要件

WebUI（Next.js）で以下を確認可能にする：

- 現在のポジション・有効証拠金・未実現損益
- AIエージェントの議論ログ（チャット風UI）
- 取引履歴と各判断の根拠
- 基本設定（リスク率・レバレッジ等）の変更

| Phase | 内容 |
